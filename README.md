# Walkamolee's Claude Code Usage

Public log of my [Claude Code](https://claude.com/claude-code) token usage and API-equivalent cost, generated by [ccusage](https://github.com/ryoppippi/ccusage).

## What's in here

Reports are organized **per machine** so the 4 computers I use Claude Code on don't clobber each other:

- `reports/<hostname>/YYYY-MM-DD-daily.json` — per-day token totals and costs across all models, generated by `ccusage daily --json`.
- `reports/<hostname>/YYYY-MM-DD-monthly.json` — same totals rolled up by month, from `ccusage monthly --json`.
- `scripts/snapshot.sh` — drops a fresh dated snapshot into `reports/<hostname>/`.
- `scripts/auto-push.sh` — cron-safe wrapper that runs the snapshot, commits, and pushes (with pull-rebase + retry so parallel machines don't collide).

Each snapshot reflects all Claude Code sessions logged under `~/.claude/projects/` on that machine at the moment it ran.

## What is NOT in here

By design, these reports do not include prompts, responses, project folder names, or file paths. They are aggregate counters only. I do not commit `ccusage session` output — that format includes per-session project paths.

## A note on the dollar figures

`totalCost` is calculated by applying public Claude API prices to the logged tokens. If you're on a Claude Max / Pro subscription, those numbers are *what this usage would cost at API rates*, not what actually gets billed. Token counts are exact; dollars are a yardstick, not an invoice.

## Regenerating the reports

```bash
# one-off for the current machine
./scripts/snapshot.sh

# full pipeline (snapshot + commit + push), what cron runs
./scripts/auto-push.sh
```

Install ccusage globally with `npm i -g ccusage`.

## Daily cron (per machine)

Each machine runs its own crontab entry:

```cron
55 23 * * * /home/<user>/projects/walkamolees-claude-code-usage/scripts/auto-push.sh >> $HOME/.cache/ccusage-cron.log 2>&1
```
